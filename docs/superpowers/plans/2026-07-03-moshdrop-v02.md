# moshdrop v0.2 实现计划（审计驱动）

> 执行者：本会话内联 TDD。范围 = 用户拍板的 ①-⑤ 全做 + 3 个默认必修 bug。
> 依据：docs/superpowers/audit-2026-07-03.md（40 确认缺陷）。

**Goal:** 让 fail-open 承诺变真、上传通道不传错机器、大文件体面、失败说人话、留痕可诊断。

## 任务与关键设计决策

### T1 fail-open 真实化（审计 A1/A2/A4/D2）
- target 为空 → Uploader 进入 disabled 态，intercept 直接跳过 → 真·纯透传（零吞秒）。
- 干掉一次性 `ready` channel + 粘死错误：改为 `ensure(ctx)`——成功缓存、失败不缓存，每次 Upload 按需重试建连。
- 退出时 `ssh -O exit` 收掉 ControlMaster（不留 120s 残连）。

### T2 上传通道跟随 mosh 参数（C1/C2）
- `DeriveSSHTarget` → 返回 (target, sshArgv)：识别 mosh 全部取值长选项的 `--opt value` 两段式（--ssh/--predict/--port/--family/--client/--server/...），修"选项值被当主机名"。
- 解析 `--ssh="ssh -p 2222"`，其 argv 同时作为上传通道基座——端口/跳板与 mosh 完全一致，杜绝传错机器。

### T3 退出路径无害化（B5）
- stdin EOF 不再注入双 ^D（会误杀 tmux 前台进程）；改为直接 SIGHUP 子进程（等价于真实终端关闭语义）。

### T4 输入热路径加固（B1/B2/B3/B7/B8）
- **单锁序列化**：Feed/Idle 产出事件与写 ptmx 在同一临界区（消灭字节乱序竞态）；异步注入共用同一把锁。
- **stat 出热路径**：粘贴先按语法（绝对路径 token）暂扣，异步 stat（500ms 硬上限，goroutine+select）；非文件/超时 → 立即原样回放。网络挂载盘再也冻不住终端。
- SIGTERM/SIGHUP 处理：恢复终端态 → 转发信号给子进程 → 退出。
- 子进程退出后排空 ptmx 再返回（不丢最后一屏）。

### T5 载荷与远端脚本正确性（C3/C4/D1）
- 远端命令统一包 `sh -c '<script>'`（fish/csh 远端可用；`'\''` 转义 fish/csh 兼容）。
- 回显文件名以 `printf '%s\n'` 结尾、本端仅剥一个尾 `\n`（不再 TrimSpace 毁首尾空白名）。
- 载荷含未转义 `\n`/`\r` → 直接判非拖拽放行（换行分隔列表不再被劫持改写）。

### T6 失败说人话（②，C6）
- 捕获 ssh stderr，分类翻译：网络不通 / 密钥拒绝 / 磁盘满 / 超时 / 其他（附原文摘要）。通知与日志共用。

### T7 大文件体面（③，A3/B4/C5）
- 自适应超时：max(60s, 按 100KB/s 估算)；失败静默重试一次（重建连接）。
- 远端先写临时名 `.part-*`，成功后 `mv` 正式名（半截文件绝迹）。
- >8MB 弹"正在上传 (xx MB)"，完成弹"已送达"。
- 上传队列串行化（单 worker）：注入顺序 = 拖拽顺序；同名并发互相截断消失。

### T8 事件日志（④）
- `~/.moshdrop/events.log` JSONL：ts/target/files/bytes/ms/结果/错误类别；1MB 轮转。

### T9 doctor + CLI 门面（⑤，D4）
- `moshdrop doctor [host]`：mosh 版本 → 目标推导回显 → ssh 连通+延迟 → 远端目录可写 → 探针文件端到端（sha256 校验）→ ControlPath 长度 → 解读最近失败日志。逐项 ✓/✗ + 修复建议。复用 Uploader 真实链路。
- `--help` / `--version`。

### T10 测试真断言化 + 开源卫生（E1-E5, D3）
- proxy 测试改 `sh -c 'stty raw -echo; exec cat'`：输出=精确字节，断言相等与顺序。
- 修 sanitizeName 假用例；补括号标记保留断言；补 2MiB 溢出与 bare 启发式覆盖；E2E 主机改 `MOSHDROP_E2E_HOST` env。
- LICENSE(MIT)、.gitignore 补 .DS_Store 并从 git 清除、`go mod tidy`。

### T11 终验
- 全量测试 + 真机 e2e + RTT 基准回归 ≤ 基线+0.5ms；multi-agent 对抗审查 v0.2 diff；版本号 0.2.0；合并推送。
