# moshdrop v0.1 全维度审计报告

日期：2026-07-03　方法：123 智能体多维审计（6 维并行审读 → 每发现 2 名对抗验证者 → 3 视角功能提案 + 产品评审 → 完整性批评家兜底）
原始发现 56 → **确认 40 / 存疑 6 / 击毙 10**（击毙 = 被反驳者证明实际不成立）

## 总体鲁棒性结论

- **快乐路径坚实**：透传保真、拦截验真、上传注入全链路经受住对抗验证；无 high 级缺陷存活（所有报 high 项均被双重验证降级 medium）。
- **最大主题缺陷：fail-open 的承诺在多条路径上是假的**——失败时不是"立即放行"，而是吞 90 秒才吐回，且 Prewarm 一次失败会让整个会话终身丧失拖拽能力。
- **测试套件有系统性假通过**：Contains+sleep 式断言测不出乱序/丢字节；多个"覆盖"实际从未触达目标分支。

## 确认缺陷（按根因聚类，40 项去重后 ≈ 22 个根因）

### 簇 A：fail-open 破洞（修复优先级最高）
| # | 位置 | 问题 | 验证严重度 |
|---|---|---|---|
| A1 | main.go:30 | target 推导失败时 `u.ready` 永不关闭：拦截照常吃掉粘贴，阻塞 90s 才吐回（"降级纯透传"并未实现）。触发：`moshdrop -- host` 等 | med/med ×4 维度独立发现 |
| A2 | uploader.go:49 | Prewarm 一次性且错误永久缓存：启动瞬间断网 → 整个会话（可活数天）拖拽永久报废，无重试 | med/med ×2 |
| A3 | uploader.go:101 | 90s 固定总超时判死 GB 级文件；失败后远端遗留截断半截文件，永不清理且污染重名判定 | med/med |
| A4 | proxy.go:74 | fail-open 重放最迟延后 90s，把过期粘贴注入用户"当下"的任意上下文（可能已切到别处） | med/med |

### 簇 B：输入通路并发与顺序
| # | 位置 | 问题 | 验证严重度 |
|---|---|---|---|
| B1 | proxy.go:104 | Idle 冲刷在 scMu 外写 ptmx，与 stdin 循环竞态 → 字节乱序（如 vim 的 ESC 与后续键对调）。窗口极窄但真实 | med/low ×3 维度 |
| B2 | proxy.go:123 | `os.Stat` 同步跑在 stdin 循环里：拖入半死 SMB 挂载点文件 → stat 挂起 → **整个终端冻结**（Ctrl-C 也进不去） | med/med ×2 |
| B3 | proxy.go:82 | 异步注入可插进"被拆分转义序列"中间，或落进 >2MiB 溢出后远端未闭合的粘贴内 | med/low |
| B4 | proxy.go:73 | 注入落点不受控：上传期间切 tmux 窗格/进 vim → 路径打进错误位置；连续拖拽注入顺序=完成顺序而非拖拽顺序 | med/low |
| B5 | proxy.go:146 | stdin 关闭时主动注入双 ^D：可能杀死 tmux 里的远端前台进程（裸 mosh 不会这样） | med/med |
| B6 | scanner.go:93 | 单独 ESC 键固定延迟 50–75ms 才送达（partialSuffix 扣留 + Idle 周期） | low/low |
| B7 | proxy.go:47 | 无 SIGTERM/SIGINT 处理：外部 kill 后终端滞留 raw 模式、mosh 变孤儿 | low/low |
| B8 | proxy.go:53 | 子进程退出后不排空 ptmx，最后一屏输出可能丢失 | low/low |

### 存疑（PLAUSIBLE，验证者意见分裂，记录备查）
- 非括号后备假设"整块单次读入"，跨 read 到达时误传前缀文件（prot）
- 纯文本换行路径列表被当拖拽劫持（prot）
- Notify osascript 挂起可扣押 fail-open 重放（robu）
- SIGWINCH goroutine 泄漏（qual，一次性泄漏，影响可忽略）
- 目录拖拽静默放行无解释（ux）
- 无 README/--help/--version 的首次体验缺口（ux）

## 功能路线图（3 视角提案 → 产品评审合并）

### v0.2（修 bug 优先，随后按序）
1. **修复 fail-open 破洞**：无 target 立即直通；Prewarm 错误去黏性 + 按需重试 [S]
2. **失败可自解释**：透出 ssh 真实 stderr + 错误分类 [S]
3. **上传反馈与超时治理**：进行中/完成通知 + 按大小自适应超时 + 远端临时名写入成功后 mv（消灭半截文件） [M]
4. **JSONL 事件日志**：每次拖放/拒绝留痕（仅元数据） [S]
5. **doctor 子命令**：全链路逐项体检 + 修复建议 [M]

### v1.0（开源发布版）
- 英文优先文案 [S]；--version/--help + goreleaser（双架构/签名/公证） [M]
- 远端 drop 目录 TTL 自动清理（默认 7 天可配） [S]
- 按 host 配置 ~/.moshdrop/config.toml [M]
- 文档化 wrap-anything 模式（et/ssh 也能包） [S]
- 剪贴板截图直喂（keybind 触发 pbpaste 上传） [M]
- 大文件护栏 + 可选限速 [M]；注入防错位（高危落点改走剪贴板+通知） [M]
